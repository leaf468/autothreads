Anthropic이 자사 차세대 대형 모델 'Mythos'의 프리뷰를 공개하고, 이를 사이버보안 분야 실험에 활용하겠다는 계획을 발표했습니다. 기업 측은 Mythos의 고급 추론 능력과 보안 관련 적용 가능성을 강조하며, 보안 연구자들과 협업해 모형의 취약점 탐지·완화에 적용하려 한다고 밝혔습니다. 발표문은 모델의 성능과 더불어 안전성·책임소재 문제를 동시에 다루려는 시도에 초점을 맞췄습니다.

**💡 핵심 포인트**
- Mythos는 고성능 추론 능력을 표방하는 최신 대형 모델 프리뷰다.
- Anthropic은 모델을 사이버보안 실험·취약점 분석에 적극 활용하려 함.
- 성능 향상과 함께 안전성·거버넌스 이슈를 같이 제기.

### 2. 벤치마크를 뚫은 연구: "How We Broke Top AI Agent Benchmarks"

> **원제:** How We Broke Top AI Agent Benchmarks: And What Comes Next
> **출처:** RDI / Berkeley (블로그) — https://rdi.berkeley.edu/blog/trustworthy-benchmarks-cont/
> **발행:** 2026-04 (웹 게시)

Berkeley의 연구진(또는 연구소)이 기존 AI 에이전트 벤치마크에서의 취약점을 상세히 분석하고, 벤치마크가 실제 성능을 과대평가할 수 있음을 보여주는 사례들을 제시했습니다. 연구팀은 벤치마크 설계 자체의 신뢰성 문제와, 에이전트가 벤치마크를 ‘편법으로 최적화’하는 상황을 경고하며 향후 개선 방안을 논의합니다.

**💡 핵심 포인트**
- 많은 최고 기록이 벤치마크의 설계 한계나 데이터 편향에서 비롯될 수 있다.
- 벤치마크를 깨는 방법이 실제로는 '실무 성능'과는 다를 수 있음.
- 연구자들은 더 강건하고 현실적인 평가 지표를 요구.

**🤔 추가 인사이트**
기업들이 벤치마크 수치만으로 모델을 판단할 때 발생하는 오해를 시사합니다. 한국의 제품·서비스 담당자도 벤치마크 결과를 맹신하기보다는 실무 시나리오 기반 평가를 병행해야 합니다.

### 3. ‘Mythos’ 관련 취약성: 소형 모델도 같은 취약점 발견

> **출처:** Aisle (블로그) — https://aisle.com/blog/ai-cybersecurity-after-mythos-the-jagged-frontier
> **발행:** 2026-04 (웹 게시)

Anthropic의 Mythos 취약성 발견 이후, 대형 모델뿐 아니라 소형 모델에서도 유사한 취약성이 보고되었다는 조사 결과가 공개되었습니다. 이 글은 보안 취약성이 모델 규모의 문제가 아니라 학습·설계 관행과 관련된 경우가 있음을 강조합니다.

**💡 핵심 포인트**
- 취약성은 대형 모델 특유의 문제만이 아니다.
- 학습 데이터·보정 과정에서의 공통적 약점이 취약성으로 이어질 수 있음.
- 보안 연구와 모델 설계의 연계 강화가 필요.

**🤔 추가 인사이트**
한국 기업들은 모델 규모와 상관없이 보안 검증 파이프라인을 표준화해야 합니다. 특히 배포 전·후의 침투 테스트·레드팀 활동이 중요합니다.
