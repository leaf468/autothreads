## 📰 주요 뉴스 (3개)

### 1. 에이전트 생태계의 전환 — “All your agents are going async”

> **원제:** All your agents are going async
> **출처:** Zak Knill (/dev/knill) (https://zknill.io/posts/all-your-agents-are-going-async/)  
> **발행:** 2026-04-20

전통적 챗형 상호작용(동기 호출 + 스트리밍 응답)이 아니라, 에이전트가 백그라운드에서 장시간·비동기적으로 동작하는 패러다임 전환을 주장하는 글입니다. 예약 작업, 웹후크, 원격 제어, 멀티-디바이스/멀티-휴먼 세션 등 에이전트 수명주기가 HTTP 요청-응답 모델에 적합하지 않다는 점을 지적합니다. Anthropic, OpenClaw, ChatGPT Tasks 등 여러 플랫폼의 비동기 기능과 함께, 세션 상태 보존과 내구성 있는 전송(Durable transport)이 핵심 문제로 부상하고 있다고 정리합니다.

대표 사례 및 흐름을 정리하면 다음과 같습니다.

**💡 핵심 포인트**
- 에이전트는 더 이상 단일 HTTP 세션에 종속되지 않음 — 백그라운드·예약·원격 동작 확대
- 세션 상태의 내구성(대화/작업 상태 저장)과 내구성 있는 전송(푸시·푸시-서브스크라이브)이 새로운 설계 요구
- Anthropic/Cloudflare는 상태 관리(Hosted session)에 집중, Ably 등은 "내구성 있는 전송" 솔루션 제시

**🤔 추가 인사이트**
한국 기업은 에이전트 도입 시 "세션 저장소(로그·콘텍스트 보존)"와 "알림/푸시 인프라" 설계를 우선 검토해야 합니다. 비동기 에이전트는 권한·보안(승인 워크플로우, 신원 연계)과 비용(장시간 작업의 토큰/컴퓨트 과금) 관점에서도 기존 챗봇과 다른 운영 패턴을 요구합니다.

### 2. CrabTrap — 에이전트를 위한 LLM 판사(오픈소스) 등장

> **원제:** CrabTrap: An LLM-as-a-judge HTTP proxy to secure agents in production
> **출처:** Brex / CrabTrap (https://www.brex.com/crabtrap)  
> **발행:** 2026 (프로젝트/오픈소스 공개)

CrabTrap은 에이전트가 외부로 보내는 모든 HTTP 요청을 가로채 정책 기반 또는 LLM 판단으로 허용/차단하는 프록시입니다. 실시간 정책 적용과 결정 로그(허용·차단 이유)를 남기며, 정적 룰과 LLM 기반 심사(heuristic judgment)를 조합해 에이전트의 오작동·데이터 유출 위험을 낮추려는 목적입니다. 오픈소스로 공개되어 즉시 실험·통합이 가능합니다.

**💡 핵심 포인트**
- 에이전트의 모든 외부 호출을 중앙에서 검토(실시간 차단/허용)
- 정규 표현식·정책 룰 + LLM 심사 조합으로 휴리스틱 보강
- 운영 시 감사 로그·결정 근거(why)를 남겨 컴플라이언스에 유리

**🤔 추가 인사이트**
국내에서 에이전트를 생산환경에 도입할 때 CrabTrap와 같은 미들웨어는 빠르게 표준이 될 가능성이 큽니다. 특히 민감한 API 호출(개인정보·결제·외부 DB 접근)에 대해 권한 상승 없이 자동 호출이 발생하는 것을 방지할 수 있어, 법적·보안적 요구사항 준수에 도움이 됩니다.

### 3. Kuri — 에이전트용 경량 브라우저 자동화 도구(토큰 효율 강조)

> **원제:** Kuri — Browser automation and web crawling for AI agents (GitHub)
> **출처:** justrach/kuri (https://github.com/justrach/kuri)
> **발행:** 2026 (레포·릴리스 활동 중)

Kuri는 에이전트 루프(페이지 스냅샷 → 액션 → 스냅샷)를 위해 설계된 경량 브라우저 자동화 도구입니다. 토큰 비용 절감(스냅샷 최적화), 작은 바이너리(배포 용이성), QuickJS 기반의 서버사이드 JS 실행 등으로 "에이전트 관점"에서의 웹 상호작용을 최적화한 점이 특징입니다. CDP 기반 모드와 "kuri-fetch" 같은 노-브라우저 모드로 유연하게 동작합니다.

**💡 핵심 포인트**
- 에이전트 워크플로우 대비 토큰 효율 최적화(스냅샷 포맷·필터링)
- 경량 바이너리(배포·엣지 배치 용이)와 standalone fetch 모드 제공
- 보안·스텔스(봇 탐지 회피) 기능 내장으로 실무 적용성 고려

**🤔 추가 인사이트**
한국의 모니터링·크롤링·자동화 에이전트 구축 시 Kuri 같은 도구는 비용·성능 관점에서 매력적입니다. 특히 모델 호출 비용(토큰)과 브라우저 인스턴스 운영비를 동시에 줄여야 하는 팀에게 우선 검토 대상입니다.

## 🔍 마이너 발견 (3개)

### 1. 에세이: "Context Is Software, Weights Are Hardware"

> **원제:** Context Is Software, Weights Are Hardware
> **출처:** Aravind Jayendran (https://www.aravindjayendran.com/writing/context-is-not-learning)  
> **발행:** 2026-04-18

컨텍스트(키/값 캐시)를 소프트웨어, 가중치(모델 파라미터)를 하드웨어에 비유하며 두 메커니즘의 역할과 한계를 분석합니다. 인컨텍스트 러닝이 가중치 업데이트의 일부 효과를 흉내낼 수 있지만, 지속성·효율성·컴포저빌리티 측면에서 가중치 변경(학습)의 장점이 있다고 주장합니다. 장기적 관점에서 "컨텍스트 확장 + 가중치 학습 병행" 전략을 제안합니다.

왜 주목할까?
- 모델 운영 전략(긴 맥락 vs. 파인튜닝)의 비교 기준을 명확히 제시
- 기업의 영속적 지식(도메인 적응)에 대한 설계 방향(컨텍스트로 버티기 vs. 모델 업데이트) 판단에 도움

### 2. MuJoCo 3.7.0 릴리스(연관: 시뮬·로봇·시각화 연구)

> **원제:** MuJoCo release 3.7.0
> **출처:** google-deepmind/mujoco (https://github.com/google-deepmind/mujoco)  
> **발행:** 2026-04-14 (release)

MuJoCo의 새 릴리스는 시뮬레이션 성능·플랫폼 지원 및 Python 바인딩 개선을 포함합니다. 로봇 공학·강화학습 연구에서의 시뮬레이터 업데이트는 시뮬 기반 학습 파이프라인(데이터 수집·시뮬-리얼 전이)에도 직접적 영향을 미칩니다.

왜 주목할까?
- 로봇·시뮬레이션 기반 AI 연구/제품팀의 인프라(테스트·시뮬) 안정성 향상
- 시뮬 성능 개선은 샘플 효율·연구 반복 속도에 직결

### 3. 내구성 있는 전송 솔루션(Ably 등)의 부상 — 에이전트 인프라의 또 다른 축

> **출처:** Zak Knill 글 내 참조(Ably) (https://ably.com/ai-transport)

에이전트의 비동기·장기 실행 문제를 해결하기 위해 Ably 같은 실시간 메시징/전송 플랫폼이 "세션 기반의 내구성 있는 전송"을 제안하고 있습니다. 단순한 HTTP 폴링 대신에 멀티-디바이스/멀티-유저를 아우르는 내구 세션을 제공하면 에이전트의 알림·결과 전달·세션 복원성이 개선됩니다.

왜 주목할까?
- 에이전트 제품화 과정에서 단순 API 게이트웨이 외에 전송층 설계가 중요해짐
- 국내 서비스도 실시간 메시징·푸시 아키텍처 재검토 필요
