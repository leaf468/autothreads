# LinkedIn Post Writing Skill (2026)

LinkedIn은 전문 비즈니스 네트워킹 플랫폼으로, 12억 명 이상의 회원을 보유하고 있습니다. 전문성과 진정성을 중시하는 플랫폼 특성을 반영해야 합니다.

## 핵심 원칙

### 1. 어투와 톤
- **전문적이지만 접근 가능한**: 격식 있되 딱딱하지 않게
- **진정성 중심**: 개인의 실제 경험과 인사이트 공유
- **교육적**: 배운 점, 인사이트 전달
- **겸손하되 자신감 있게**: 과도한 자기PR 지양
- **존댓말 사용**: 한국어의 경우 친근한 존댓말

### 2. 글자 수 및 구조
- **최적 길이**: 150-300자 (한국어 기준)
- **최대**: 3,000자까지 가능하지만 간결할수록 좋음
- **첫 두 줄**: 가장 중요 (미리보기에 노출)
  - 핵심 메시지를 첫 두 줄에 담기
  - 스크롤을 멈추게 하는 훅 제공

### 3. 문장 구조

```
[강력한 오프닝 문장]
핵심 메시지나 흥미로운 질문

[본문: 2-4개 단락]
- 각 단락은 1-2문장
- 명확한 줄바꿈으로 가독성 확보
- 구체적인 사례와 숫자 활용

[마무리]
핵심 인사이트 요약 또는 독자 참여 유도 질문
```

## 콘텐츠 스타일

### 효과적인 포스트 유형
1. **경험 공유**: 실제 겪은 일에서 배운 교훈
2. **인사이트 제공**: 업계 트렌드, 전문 지식
3. **스토리텔링**: 실패담, 성장 과정
4. **How-to 가이드**: 실용적인 조언과 팁
5. **질문 던지기**: 토론을 시작하는 질문

### 피해야 할 것
- ❌ 과도한 자기 자랑
- ❌ 추상적이고 막연한 조언
- ❌ AI 생성티 나는 완벽한 글
- ❌ 너무 긴 문단 (가독성 저하)
- ❌ 판매 중심 메시지

## 해시태그 사용법 (2026년 중요 변화)

**중요**: LinkedIn에서 해시태그는 더 이상 콘텐츠 노출에 큰 영향을 주지 않습니다.

- **권장**: 꼭 필요한 경우가 아니면 생략
- **사용 시**: 1-2개의 관련성 높은 해시태그만
- **위치**: 포스트 끝에 자연스럽게
- **과다 사용 금지**: 3개 이상은 스팸으로 인식될 수 있음

예시:
```
#AI에이전트 #디지털전환
```

## 이모지 사용

- **최소한 사용**: 전문성 유지를 위해 절제
- **허용되는 경우**:
  - ✅ 체크 포인트 강조
  - 💡 인사이트 표시
  - 🎯 핵심 메시지 강조
  - 📊 데이터 관련 내용
- **개수**: 포스트당 1-3개 정도

## 게시 최적화

### 게시 빈도
- **권장**: 주 2-5회
- **효과**: 주 2회 이상 게시 시 프로필 조회수 5배 증가

### 영상 콘텐츠
- 일반 게시물 대비 **1.4배 높은 반응**
- 공유 가능성 **20배 증가**
- 자막 삽입 권장 (소리 없이 시청)

### 참여 유도 전략
1. **질문으로 마무리**: 댓글 유도
2. **첫 댓글 활용**: 추가 정보나 링크 첨부
3. **48시간 내 댓글 응답**: 알고리즘 부스트

## 포스트 작성 프로세스

### STEP 1: 리서치 (필수)
- `researcher` 서브에이전트 활용
- 최소 3개 이상의 신뢰할 수 있는 출처
- 구체적인 데이터와 사례 수집

### STEP 2: 구조 설계
```
1. 훅 (첫 두 줄)
   - 궁금증 유발하는 문장
   - 또는 강력한 인사이트

2. 맥락 제공
   - 왜 이 이야기를 하는가
   - 독자에게 왜 중요한가

3. 핵심 내용
   - 구체적인 사례
   - 숫자와 데이터
   - 실제 경험

4. 마무리
   - 핵심 인사이트 정리
   - 독자 참여 유도 질문
```

### STEP 3: 작성
- 짧은 문장, 명확한 단락
- 전문 용어는 설명과 함께
- 능동태 사용
- 구체적인 예시 포함

### STEP 4: 검토
- [ ] 첫 두 줄이 강력한가?
- [ ] 구체적인 사례가 있는가?
- [ ] 진정성이 느껴지는가?
- [ ] 독자가 행동할 이유가 있는가?
- [ ] 가독성이 좋은가? (단락 구분)

## 성공적인 LinkedIn 포스트 예시

### 예시 1: 경험 공유형

```
3년 전 첫 프로젝트 관리를 맡았을 때, 저는 큰 실수를 했습니다.

모든 것을 혼자 해결하려 했죠. 결과적으로 프로젝트는 2주 지연되었고, 팀원들과의 관계도 소원해졌습니다.

그 경험에서 배운 3가지:

✅ 위임은 약점이 아닌 강점입니다
✅ 명확한 커뮤니케이션이 완벽한 계획을 이깁니다
✅ 팀의 다양한 관점이 혁신을 만듭니다

여러분은 리더십에서 가장 중요한 교훈이 무엇이라고 생각하시나요?

#리더십
```

### 예시 2: 인사이트 제공형

```
2026년 마케팅 예산의 65%가 AI 기술에 투자되고 있습니다.

하지만 성공한 캠페인을 분석해보니 공통점은 기술이 아니었습니다.

성공 요인 TOP 3:
1. 고객에 대한 깊은 이해 (85%)
2. 진정성 있는 스토리텔링 (78%)
3. 일관된 브랜드 경험 (72%)

AI는 도구입니다. 결국 사람의 마음을 움직이는 것은 사람입니다.

여러분의 경험은 어떠신가요?

#마케팅 #AI
```

### 예시 3: How-to 가이드형

```
주니어 개발자 면접에서 가장 많이 받는 질문:
"실패한 프로젝트에 대해 말씀해주세요"

잘못된 답변:
"저는 실패한 적이 없습니다"

좋은 답변 구조:
1. 상황 설명 (간략하게)
2. 무엇을 배웠는지 (구체적으로)
3. 이후 어떻게 적용했는지 (결과 포함)

면접관이 보고 싶은 건 완벽함이 아닌 성장 가능성입니다.

취업 준비 중이신 분들, 화이팅입니다! 💪
```

## 파일 저장 형식

포스트 작성 후 다음 경로에 저장:
```
linkedin/<slug>/post.md
```

slug는 주제를 나타내는 짧은 영문 (예: ai-agents, project-management)

## 이미지 생성 (선택사항)

LinkedIn 포스트는 이미지가 선택사항이지만, 있으면 참여도가 높아집니다.

이미지가 필요한 경우 `generate_social_image()` 도구 사용:

```python
generate_social_image(
    prompt="전문적이고 깔끔한 디자인. 파란색과 회색 톤의 그라데이션 배경에 연결된 노드들이 네트워크를 형성하는 모습. 중앙에 빛나는 아이콘. 텍스트 없음. 비즈니스 전문가를 위한 세련된 이미지.",
    platform="linkedin",
    slug="ai-agents"
)
```

이미지 프롬프트 가이드라인:
- **톤**: 전문적, 세련됨, 깔끔함
- **색상**: 파란색, 회색, 흰색 계열 (LinkedIn 브랜드 톤)
- **구성**: 심플하고 명확한 메시지
- **텍스트**: 이미지에 텍스트 넣지 않기 (플랫폼이 추가)

## 체크리스트

포스트를 게시하기 전에 확인:

- [ ] 포스트가 `linkedin/<slug>/post.md`에 저장되었는가?
- [ ] 이미지가 필요한 경우 생성되었는가?
- [ ] 첫 두 줄이 스크롤을 멈추게 하는가?
- [ ] 구체적인 사례나 데이터가 포함되었는가?
- [ ] 진정성이 느껴지는가? (AI 생성티가 나지 않는가?)
- [ ] 단락이 명확히 구분되어 읽기 쉬운가?
- [ ] 독자 참여를 유도하는 질문이나 CTA가 있는가?
- [ ] 해시태그가 1-2개로 제한되었는가?
- [ ] 전문성을 유지하면서도 접근 가능한 톤인가?

## 2026년 LinkedIn 알고리즘 핵심

1. **진정성**: AI 생성 콘텐츠보다 실제 경험담 우대
2. **참여도**: 댓글, 공유가 좋아요보다 중요
3. **영상 우선**: 네이티브 영상 콘텐츠 노출 증가
4. **커뮤니티**: 친밀도 높은 연결에 우선 노출
5. **체류 시간**: 얼마나 오래 읽는가가 핵심 지표
