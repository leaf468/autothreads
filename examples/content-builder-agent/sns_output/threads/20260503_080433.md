### 1. IBM, Granite 4.1 모델군 공개 — 엔터프라이즈용 멀티모달·저비용 파운데이션 모델 세트

> **원제:** Introducing the IBM Granite 4.1 family of models
> **출처:** IBM Research — https://research.ibm.com/blog/granite-4-1-ai-foundation-models
> **발행:** 2026-04-29

IBM Research가 언어·비전·음성·임베딩·가드(Guardian) 모델을 포함하는 Granite 4.1 컬렉션을 공개했습니다. 모델 크기는 3B/8B/30B급을 포함하고, instruction‑following과 도구 호출(tool-calling) 성능을 중점으로 설계했다고 합니다. 특히 8B급 instruct 모델이 이전 세대나 동급 모델 대비 높은 효율을 보였고, 일부 8B 모델이 32B Mixture‑of‑Experts 모델과 성능을 견줄 만큼 개선되었다고 발표했습니다.

Granite Vision은 표·차트·KVP(키‑값) 추출에 강점을 둔 VLM으로 엔터프라이즈 문서 이해에 최적화되어 있고, Granite Speech는 다국어 ASR/번역에서 우수한 WER(예: 2B 모델 5.33% WER)에 도달해 엣지 환경까지 타깃으로 삼습니다. Guardian(감시) 모델은 안전·편향·환각(hallucination) 탐지 기능을 담당해 파이프라인 내에서 리스크를 사전 차단하는 역할을 합니다.

흥미로운 기술적 포인트는 학습 철학입니다. IBM은 단순 토큰량 경쟁보다 데이터 품질과 단계적 정제(staged refinement)를 택해 약 15조 토큰을 단계적으로 고품질 데이터로 anneal 했고, 마지막 단계에서 최대 512K 토큰의 긴 컨텍스트를 지원하도록 확장했다고 밝힙니다. 또한 모델과 데이터·체크포인트는 Apache 2.0 라이선스로 공개되어 있어 기업 도입·온프레미스 운영에 유리합니다.

**💡 핵심 포인트**
- 멀티모달(언어·비전·음성) 모델을 묶은 엔터프라이즈 지향 컬렉션
- 8B급과 같은 상대적으로 작은 모델에서 높은 실무 성능(지연·비용 측면 우위) 확보
- Granite Guardian으로 파이프라인 내 안전·정책 검증을 모델 단에서 제공
- Apache 2.0 공개 라이선스 + vLLM/llama.cpp 등 오픈 추론 런타임 호환

**🤔 추가 인사이트**
한국 기업 관점에서는 ‘대형 폐쇄 모델 의존도를 낮추고, 비용·규제·온프레 요구를 맞춘 실무적 대안’이 될 가능성이 큽니다. 특히 문서 자동화(회계·인보이스·보고서)나 콜센터 전사·요약 같은 정형화된 엔터프라이즈 워크플로우에선 Granite Vision/Speech 조합이 실효성을 가질 수 있습니다. 운영 관점에서는 Apache 2.0 공개와 로컬 추론 최적화(경량화·512K 컨텍스트 지원)가 도입 결정에 긍정적 요소입니다.

### 2. DeepMind 출신 데이빗 실버, 'Ineffable Intelligence'로 11억 달러 규모 펀딩 유치 — RL 기반 슈퍼러너 목표

> **원제:** DeepMind's David Silver just raised $1.1B to build an AI that learns without human data
> **출처:** TechCrunch — https://techcrunch.com/2026/04/27/deepminds-david-silver-just-raised-1-1b-to-build-an-ai-that-learns-without-human-data/
> **발행:** 2026-04-27

DeepMind의 핵심 연구자였던 David Silver가 설립한 영국 기반 스타트업 'Ineffable Intelligence'가 약 11억 달러(약 1.1B USD) 규모의 자금을 유치했습니다. 회사는 인간 데이터에 의존하지 않고 강화학습(RL) 기반으로 스스로 지식과 스킬을 발견하는 ‘superlearner’를 표방합니다. 투자에는 Sequoia, Lightspeed 등 벤처와 Nvidia·Google 등 전략적 투자자가 참여한 것으로 보도됩니다.

실버의 연구 배경(AlphaZero 등)과 RL 전문성에 기대어, 이 라운드는 ‘RL로 인간 데이터 의존도를 낮추려는 실험’에 큰 베팅을 한 사례로 해석됩니다. Ineffable은 대규모 시뮬레이션·자체 환경을 활용해 모델이 경험을 통해 학습하도록 하는 접근을 강조합니다. TechCrunch 보도는 또한 런던/UK 쪽 AI 허브 형성 가속(DeepMind 잔류 인력·네트워크 활용)을 주목합니다.

**💡 핵심 포인트**
- 대형 자금 유치: 연구 중심의 대규모 시도에 시장이 자금을 투입
- 목표: 인간 레이블/데이터에 덜 의존하는 학습법(강화학습·자기주도 학습)
- 전략적 의미: 런던/UK가 RL·에이전트 연구의 허브로 재부상할 가능성

**🤔 추가 인사이트**
한국의 연구·산업계에는 두 가지 시사점이 있습니다. 첫째, RL·자기지도 학습에 대한 장기적 투자 기류가 강해지면 데이터 확보 중심의 경쟁 구도가 일부 해소될 수 있습니다. 둘째, 국내 스타트업·연구소는 시뮬레이션 인프라(시뮬 환경, 계산 자원)와 안전성 검증 파이프라인을 강화해 RL 기반 연구 경쟁력 확보를 준비해야 합니다. 또한 규제·윤리 이슈(자기학습 시스템의 예측 불가 행동)에 대한 거버넌스 논의도 조기에 병행해야 합니다.

### 3. 오픈-웨이트 중국 모델 Kimi K2.6, 실시간 코딩 대회에서 GPT·Claude·Gemini 제치고 우승

> **원제:** An open-weights Chinese model just beat Claude, GPT-5.5, and Gemini in a programming challenge
> **출처:** ThinkPol / AI Coding Contest coverage — https://thinkpol.ca/2026/04/30/an-open-weights-chinese-model-just-beat-claude-gpt-5-5-and-gemini-in-a-programming-challenge/
> **발행:** 2026-04-30

AI 코딩 대회(AI Coding Contest)에서 중국 스타트업 Moonshot AI의 오픈-웨이트 모델 Kimi K2.6가 실시간 프로그래밍 과제(Word Gem Puzzle)에서 GPT-5.5·Claude 등 상용 API 모델을 누르고 1위를 차지했습니다. 대회는 실시간 타임리미트와 복합적인 규칙(점수 시스템, 슬라이드 퍼즐)을 갖춘 환경에서 모델의 ‘실행·인터페이스 코드 작성 능력’과 ‘실시간 전략’ 능력을 테스트했습니다.

보도에 따르면 Kimi는 적극적인(aggressive) 슬라이딩 전략을 반복 적용해 지속적으로 포인트를 확보했고, 이는 오히려 더 보수적인(안전성 우선) 튜닝 모델보다 더 높은 점수를 가져왔습니다. 이번 사례는 특정 작업에서 ‘오픈-웨이트 모델이 실무적 경쟁력(또는 전술적 우위)을 보일 수 있음’을 시사합니다.

**💡 핵심 포인트**
- 오픈-웨이트 모델이 특정 실전 과제에서 최상위권을 달성
- 대회는 ‘프로토콜 준수·네트워크 연결·실행 가능한 코드 작성’ 같은 엔지니어링 난제를 포함
- 모델 전략(그리디 vs 보수적)이 성능을 갈랐으며, 과제 설계에 따라 순위가 달라질 수 있음

**🤔 추가 인사이트**
한국 기업·연구자는 오픈-웨이트 모델의 빠른 접근성과 성능 향상을 주목해야 합니다. 특히 로컬 배포·온프레미스 적용, 라이선스·컴플라이언스 측면에서 오픈 모델을 활용할 수 있는 기회가 늘고 있습니다. 다만 안전성·정책 준수(예: 과도한 공격적 행동)와 같은 거버넌스 리스크를 사전에 검증하는 체계도 병행해야 합니다.

## 🔍 마이너 발견 (3개)

### 1. arXiv — "LLM as Clinical Graph Structure Refiner" (EEG 발작 진단용 그래프 정제)

> **출처:** arXiv:2604.28178 — https://arxiv.org/abs/2604.28178

이 논문은 EEG(뇌파) 기반 발작(Seizure) 진단에서 그래프 구조(노드 간 엣지)를 정제하는 데 LLM을 도입하는 아이디어를 제안합니다. 전통적 상관관계 기반 그래프가 잡음으로 인해 불필요한 엣지를 많이 만들 수 있는데, LLM을 ‘엣지 정제기(edge refiner)’로 활용해 불필요·중복 연결을 제거하면 표현 학습과 진단 정확도가 개선된다는 주장입니다. 논문은 Transformer 기반 엣지 예측기 + LLM 기반 검증 단계로 구성된 2단계 프레임워크를 제시하며, TUSZ 데이터셋에서 성능 향상을 보고합니다.

왜 주목할까?
- 의료 데이터의 잡음 문제를 LLM의 문맥·설명 능력으로 보완하는 흥미로운 접근
- IJCAI 2026(수락) 등 학회적 검증을 통해 의료 AI 적용 가능성 확대
- 한국의 의료·헬스케어 AI 연구에서 다중 모달·해석 가능 모델 적용 시 영감 제공

원문: https://arxiv.org/abs/2604.28178

### 2. Specsmaxxing — acai.sh의 스펙 중심 개발(ACAI) 블로그 포스트

> **출처:** Acai.sh blog — https://acai.sh/blog/specsmaxxing

개발자 블로그 글 «Specsmaxxing»은 LLM 기반 워크플로우에서 ‘명세(spec) 중심’으로 작업하는 접근을 설명합니다. 저자는 Acceptance Criteria ID(ACID)를 도입해 요구사항을 번호화·추적하고, `feature.yaml` 같은 구조화된 스펙 포맷과 acai.sh 도구(오픈소스 CLI·대시보드)를 조합해 에이전트가 명세를 참조하고 구현·검증하도록 하는 실무 패턴을 제안합니다.

왜 주목할까?
- LLM이 코드·문서를 빠르게 생성할 때 ‘스펙 일관성’과 검증이 새로운 병목이 되므로, 명세 추적 도구의 필요성이 커짐
- 조직 단위의 품질관리(acceptance coverage)를 자동화하면 코드 생성 시대의 QA 프로세스를 재설계할 수 있음

원문: https://acai.sh/blog/specsmaxxing

### 3. GitHub: Maigret — 사용자명으로 3,000+ 사이트를 크롤하는 OSINT 도구

> **출처:** GitHub - soxoj/maigret — https://github.com/soxoj/maigret

Maigret는 사용자명(username)만으로 3,000개 이상의 사이트를 자동 검색해 계정·프로필 정보를 수집하는 오픈소스 도구입니다. Docker/웹 UI/CLI를 지원하며, 자동 리포트(HTML/PDF)·프록시·Tor 지원 등 실무적인 기능을 갖추고 있어 OSINT·보안 조사에서 활용도가 높습니다.

왜 주목할까?
- 빠른 침해용/조사용 프로파일링이 가능해 보안·컴플라이언스 관점에서 유용
- 그러나 개인정보·법적 이슈(수집·저장·전달)에 주의 필요 — 기업 사용 시 내부정책·법검토 필수

원문: https://github.com/soxoj/maigret
