### 3. arXiv: "Resample or Reroute?" — LLM 테스트타임 모델 선택 전략 제안

> **원제:** Resample or Reroute? Budget-Aware Test-Time Model Selection for Large Language Models
> **출처:** arXiv (https://arxiv.org/abs/2607.08665)  
> **발행:** 2026-07-10 (arXiv)

최근 arXiv에 공개된 논문은 LLM 운영 비용(토크나 API 호출 비용)을 고려한 테스트타임 모델 선택 전략을 제시합니다. 동일한 요청에 대해 예산 제약 하에서 여러 후보 모델을 동적으로 선택하거나 재시도(resample)하는 방식으로 품질-비용 트레이드오프를 최적화하는 방법론을 다룹니다.

실무적 의미는 명확합니다 — 대형 모델을 항상 사용하는 대신 상황에 따라 경량 모델·고성능 모델을 혼용해 비용을 절감하면서 SLA를 만족시키는 운영 전략을 설계할 수 있습니다. 특히 멀티모델 서빙, 캐싱, 리라이팅(rewriting) 파이프라인과 연동하면 즉시 적용 가능한 아이디어입니다.

**💡 핵심 포인트**
- 예산을 고려한 테스트타임 모델 선택은 운영 비용 절감 가능.
- 동적 재시도/모델 전환 전략이 응답 품질과 비용을 균형시킴.
- 기업용 AI 서비스(SaaS)에서 단계적 적용 시 큰 비용 절감 효과 기대.

**🤔 추가 인사이트**
한국의 AI 서비스 제공자는 API 비용·서빙 인프라 비용을 절감하기 위해 멀티모델 전략을 빠르게 도입해야 합니다. 또한 SLO/SLA 기반의 자동 전환 로직과 비용 모니터링 대시보드를 함께 설계하면 운영 효율을 극대화할 수 있습니다.

*원문: https://arxiv.org/abs/2607.08665*
