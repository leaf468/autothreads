> **원문 링크:** https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-5/

요약:
구글이 Gemini 3.5의 "Flash" 형태 업데이트를 공개하며 생성형 모델의 응답 지연(latency)과 멀티모달 처리 속도를 크게 개선했다고 발표했습니다. 이번 업데이트는 경량화된 추론 경로와 새로운 캐시·프라이오리티 스케줄러를 도입해 실시간 인터랙션(검색, 대화형 에이전트, 이미지 질문응답 등)에서 체감 성능을 끌어올리는 것을 목표로 합니다. 특히 검색 결과와 생성 결과를 실시간으로 결합하는 파이프라인을 강화해 ‘검색 → 생성’의 경계가 더 흐려졌습니다.

**💡 핵심 포인트**
- 레이턴시 감소와 멀티모달 처리 최적화로 실사용 인터랙션 품질이 개선
- 검색 결과(정보 신뢰도)와 생성 텍스트(자연스러움)를 결합하는 실험적 파이프라인 강화

### 3. 논문: "Formal Skill: Programmable Runtime Skills for Efficient and Accurate LLM Agents" (arXiv)

> **원제:** Formal Skill: Programmable Runtime Skills for Efficient and Accurate LLM Agents
> **출처:** arXiv (Preprint)
> **원문 링크:** https://arxiv.org/abs/2605.19604
> **발행:** 2026-05-20

요약:
해당 논문은 LLM 에이전트의 실행 환경에서 '스킬(skill)'을 프로그래밍 가능한 런타임 컴포넌트로 정의하고, 이를 통해 추론 비용 절감과 정확도 향상을 동시에 달성하는 방법을 제시합니다. 저자들은 스킬의 명세·검증·안전 경계(guardrails)를 체계화하고, 런타임에서 스킬을 동적으로 로드·교체하는 아키텍처 패턴을 실험적으로 검증했습니다. 결과적으로 고비용의 거대 모델 호출을 줄이고, 도메인 특화 스킬로 보완하여 전체 파이프라인 효율을 높일 수 있음을 보였습니다.

**💡 핵심 포인트**
- 스킬 단위의 런타임 모듈화로 추론 호출 최적화(비용 절감)
- 스킬 명세 기반의 검증으로 에이전트 안전성·재현성 향상
- 동적 스킬 오프로드/교체는 엣지-클라우드 하이브리드 배포에 유리

**🤔 추가 인사이트**
한국 기업의 에이전트 도입 사례에서는 ‘스킬 레지스트리’·‘버전 관리’·‘검증 파이프라인’을 조직 내 표준으로 채택하면 거대한 모델 의존도를 낮추면서 규제 준수(로그·감사)도 관리하기 쉬워집니다. 특히 금융·의료 같은 규제 민감 도메인에서 부분적 로컬 스킬로 책임 한계를 명확히 하는 전략이 유효합니다.
