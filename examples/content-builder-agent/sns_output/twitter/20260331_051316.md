### 1. 구글, 'Search Live' 전세계 출시로 실시간 멀티모달 검색 확장

> **원제:** Google is launching Search Live globally
> **출처:** Aisha Malik / TechCrunch (https://techcrunch.com/2026/03/26/google-is-launching-search-live-globally/)
> **발행:** 2026-03-26

구글이 ‘Search Live’ 기능을 전 세계로 확대 배포합니다. 사용자는 실시간으로 카메라·오디오·텍스트를 아우르는 멀티모달 입력으로 검색 결과를 즉시 얻을 수 있으며, 대화형으로 쿼리를 다듬고 작업을 자동화하는 경험을 제공한다고 발표했습니다. 휴대기기·웨이러블과의 결합으로 현장 검색(예: 제품 정보·번역·실시간 Q&A)이 더 자연스러워집니다.

이 변화는 검색 경험의 '대화화'·'도구화' 흐름을 가속화합니다. 단순 키워드 매칭을 넘어서, 사용자의 상황(context)을 실시간으로 반영해 액션을 제안하는 방향입니다. 검색 플랫폼이 개인 비서 수준의 상호작용을 목표로 삼으면서, 개인정보·프라이버시 설정과 연동한 거버넌스가 더 중요해졌습니다.

**💡 핵심 포인트**
- 멀티모달(카메라·오디오·텍스트) 실시간 검색 확장
- 현장(현장 장치)에서의 대화형 검색·작업 자동화 가능성 증가
- 프라이버시·데이터 보관 정책이 사용자 수용성 관건

**🤔 추가 인사이트**
한국 서비스 관점에서는 지도·상거래·관광·금융 서비스에 즉시 적용할 여지가 큽니다. 로컬화된 인덱싱과 실시간 오프라인 성능 개선(예: 온디바이스 추론) 전략이 경쟁력을 좌우할 것입니다.

### 2. Ollama, Apple Silicon에서 MLX 기반으로 빠른 온디바이스 LLM 실행 프리뷰 공개

> **원제:** Ollama is now powered by MLX on Apple Silicon in preview
> **출처:** Ollama blog / Hacker News (https://ollama.com/blog/mlx, https://news.ycombinator.com/item?id=47582482)
> **발행:** 2026-03-31

Ollama가 MLX 런타임을 활용해 Apple Silicon 칩에서 LLM·에이전트 워크로드를 가볍고 빠르게 실행하는 프리뷰를 공개했습니다. 목표는 서버 의존도를 낮추고, 개인정보·지연(latency) 민감한 애플리케이션을 온디바이스로 이전하는 것입니다. MLX 최적화로 모델 로딩·추론 속도가 개선되어 데모 수준에서 실사용 가능성을 보였습니다.

온디바이스 LLM 실행은 개인정보 보호·네트워크비용 절감·즉각 반응성이라는 장점을 가집니다. 다만 모델 업데이트·거버넌스(버전관리, 보안패치) 측면에서 새로운 운영·배포 모델이 필요합니다. Ollama의 접근은 특히 모바일·데스크톱 앱 개발자들이 서버 비용 없이 고급 AI 기능을 제공할 수 있는 길을 제시합니다.

**💡 핵심 포인트**
- Apple Silicon에서 MLX로 고속 온디바이스 추론 가능 프리뷰
- 서버 의존도 감소로 개인정보·지연 문제 개선 기대
- 배포·업데이트·보안 운영 방식 재설계 필요

**🤔 추가 인사이트**
한국의 모바일 앱·보안 업체는 온디바이스 모델을 통한 차별화된 UX를 검토해볼 만합니다. 특히 엔터프라이즈 내부 정책상 데이터 외부 반출을 제한하는 경우에 유리합니다.

### 3. 'Not Search, But Scan' — MLLM의 학술 논문 스캔·리즌(Test) 능력 벤치마크 (ICLR 채택)

> **원제:** Not Search, But Scan: Benchmarking MLLMs on Scan-Oriented Academic Paper Reasoning
> **출처:** Rongjin Li et al. / arXiv (Accepted to ICLR 2026) (https://arxiv.org/abs/2603.28651)
> **발행:** 2026-03-31

ICLR에 채택된 논문은 대규모 멀티모달 LLM(MLLM)이 학술 논문 같은 구조화된 문서에서 '검색'이 아니라 단기간에 핵심을 스캔·이해해 질문에 답하는 능력을 평가하는 새로운 벤치마크를 제안합니다. 전통적인 검색(정보 검색)과 달리, 논문 내부의 표·그림·방법론을 빠르게 해석해 결론을 도출하는 능력이 핵심 평가 지표입니다.

연구 결과는 MLLM이 표·수식·다중 모달 컨텍스트를 통합할 때 아직 취약점이 있음을 보였고, 스캔 지향 설계(특정 프롬프트·추론 루틴)가 성능을 크게 개선할 수 있음을 시사했습니다. 이는 학술·특허·의료 보고서 자동 요약·질의응답 시스템 설계에 직접적인 영향을 미칩니다.

**💡 핵심 포인트**
- 논문·보고서같은 구조화 문서에서 '스캔' 능력 집중 평가
- 표·수식·멀티모달 통합에서 MLLM의 약점과 개선 방향 제시
- 연구·의학·특허 분야 자동화 도구 설계에 적용 가능

**🤔 추가 인사이트**
한국 대학·연구소는 이 벤치마크를 활용해 자국어·한글 문서에 최적화된 평가·튜닝 파이프라인을 만들면 국제 비교 우위 확보에 도움이 됩니다. 의료·법률 문서 처리를 상용화하려면 도메인 특화 보정(finetune)과 엄격한 검증이 필요합니다.

왜 주목할까?

- 비용(추론 호출)·반응 속도 개선 기대

왜 주목할까?
