요약하자면, 모델 공급자 다원화와 에이전트·응용 서비스의 생산적 전환에 따라 ‘모델-사이 미들웨어’인 라우팅·게이트웨이 인프라가 핵심 플랫폼 경쟁력으로 부상하고 있다는 신호입니다.

**💡 핵심 포인트**
- 멀티모달·멀티모델 라우팅·품질·비용 최적화 기능 강화
- 엔터프라이즈 제어(가드레일, 스페이스, 비용관리 등) 제품화
- 전략적 투자자(인프라·데이터 플랫폼사) 참여로 기업 고객 확보 가속

### 2. Open Envelope — 에이전트 팀 정의를 위한 공개 스키마(Show HN)

> **원제:** Show HN: Open Envelope – an open schema for defining AI agent teams
> **출처:** Open Envelope (문서 / Show HN)
> **URL:** https://openenvelope.org/docs/schema/
> **발행:** 2026-05-31

Open Envelope 프로젝트는 다중 에이전트(팀) 구성을 표준화하기 위한 JSON 스키마를 공개했습니다. 스키마는 에이전트 역할·프롬프트·모델·액세스 정책, 서브에이전트 계층, 휴먼-인-더-루프 게이트, 파이프라인·스케줄 등을 선언적으로 기술할 수 있게 설계되어 있으며, SchemaStore·npm 등록으로 툴체인(에디터 자동완성·검증) 통합을 목표로 합니다.

이 시도는 에이전트 기반 워크플로우가 여러 런타임·프레임워크로 확산되는 상황에서 ‘이식 가능한 팀 정의’의 필요성에 응답하는 움직임으로 볼 수 있습니다.

**💡 핵심 포인트**
- 에이전트 팀을 선언적으로 기술하는 표준 스키마 제안
- 액세스 정책·휴먼 게이트 등 거버넌스 요소 포함으로 실전 배포 고려
- SchemaStore·npm 등으로 개발자 UX(검증·자동완성) 지원

**🤔 추가 인사이트**
한국의 대기업·금융권에서는 ‘표준화된 에이전트 정의’가 규제 준수·감사 로그·업무 프로세스 통합에 유리하게 작용할 수 있습니다. 다만 런타임 간 세부 실행 의미가 달라질 수 있어, 실무에서는 스키마 레벨의 계약(contract)과 런타임 별 변환·검증 파이프라인이 필요합니다.

### 3. arXiv: "Rotary GPU" — 제한된 GPU 메모리에서 대형 MoE 모델 로컬 실행 실험

> **원제:** Rotary GPU: Exploring Local Execution Paths for Large Mixture-of-Experts Models Under Limited GPU Memory
> **출처:** arXiv (논문)
> **URL:** https://arxiv.org/abs/2605.29135
> **발행:** 2026-05-27

해당 논문은 대형 MoE(Mixture-of-Experts) 모델을 제한된 VRAM(예: 8GB) 환경에서 로컬로 실행하기 위한 실행 전략을 탐색합니다. 저자들은 'Rotary GPU'라는 실행 접근법을 소개하며, 노트북급 RTX 4060(8GB)에서 Qwen3.6-35B급 MoE 모델을 실행해 2048 토큰 출력·실행 중 VRAM을 ~6.3GB 수준으로 유지한 사례를 보고합니다. 논문은 배포용 데이터센터를 대체하려는 목적이 아니라, 일부 기능을 엣지·오프라인 환경으로 끌어오는 ‘접근성(accessibility)’ 연구라는 점을 강조합니다.

**💡 핵심 포인트**
- 제한적 GPU 메모리에서 대형 MoE 일부 기능을 실행 가능한지 실험한 탐색적 연구
- 소비자·엣지 하드웨어에서 긴 출력(2048 토큰)을 유지하며 안정적 실행 가능성을 보임
- 배포·보안·프라이버시 관점에서 로컬 실행의 잠재적 장점 제시

**🤔 추가 인사이트**
한국 기업이 개인정보 민감도 때문에 클라우드 의존을 회피하려는 경우, 로컬/온프레미스에서 고성능 모델 일부를 돌리는 접근은 실용적 옵션이 될 수 있습니다. 다만 성능·신뢰성·보수 비용을 고려한 아키텍처 설계가 병행돼야 합니다.

### 1. LiteParse — 경량 문서 파서 (GitHub Trending)

> **출처:** GitHub
> **URL:** https://github.com/run-llama/liteparse

문서 파싱에 특화한 오픈소스 툴킷 LiteParse가 주목받고 있습니다. PDF·OCR·바운딩박스 기반의 경량 파싱을 제공하며, 로컬·WASM·Python/Node 바인딩을 지원해 에이전트의 문서 입출력 파이프라인에서 빠르게 채택될 가능성이 큽니다.

### 2. VoxCPM2 — 토크나이저 프리 TTS (OpenBMB, GitHub)

> **출처:** GitHub
> **URL:** https://github.com/OpenBMB/VoxCPM

왜 주목할까?
