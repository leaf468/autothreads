### 1. 개발자용 초경량 코딩 모델 — Microsoft의 MAI-Code-1-Flash 출격

> **원제:** Introducing MAI-Code-1-Flash
> **출처:** Microsoft AI (Microsoft 공식 블로그)
> **URL:** https://microsoft.ai/news/introducingmai-code-1-flash/
> **발행:** 2026-06-02

마이크로소프트가 GitHub Copilot과 VS Code 환경에 최적화된 경량 코딩 모델 "MAI-Code-1-Flash"를 발표했습니다. 실사용 환경(프로덕션 Copilot 허니스)에서의 학습·평가를 강조하며, 기존 벤치마크 최적화형 접근보다 개발자 워크플로우에 맞춘 설계가 핵심이라고 밝히고 있습니다. 발표문에는 Claude Haiku 4.5 대비 SWE-Bench 등 주요 코딩 벤치마크에서 더 높은 성공률과 적은 토큰 사용량을 제시해 효율성·성능 모두에서 우위를 주장합니다.

요약: MAI-Code-1-Flash는 간결한 응답과 복잡 문제에 대한 적응적 추론 배분을 통해 토큰 효율을 개선한 점을 핵심으로 내세우며, 점진적 롤아웃은 VS Code Copilot 사용자부터 시작합니다. 기업·개발팀 관점에서는 지연(Latency), 비용(토큰 사용), 개발자 생산성 개선 측면에서 관심이 큽니다.

**💡 핵심 포인트**
- 개발자 워크플로우 중심 설계로 Copilot·VS Code 통합을 우선 목표로 함
- 벤치마크 대비 토큰 효율 증가(최대 60% 토큰 절감 사례 제시)
- 프로덕션 허니스 기반 평가로 실무 적합성 강조

**🤔 추가 인사이트**
한국 개발 생태계에서는 Copilot 기반의 대체 모델이 확산되면 비용 구조(토큰 비용·응답 지연)에 따른 도입·운영 전략이 달라질 수 있습니다. 특히 SaaS형 개발도구·IDE 통합 사례에서 이용자 프라이버시·레포지토리 데이터 처리 정책을 사전에 검토해야 합니다.

### 2. 법학 교육의 충격적 결과 — 스탠포드 연구: 교수들이 AI 답변을 더 선호

> **원제:** AI Outperforms Law Professors in Stanford Law Study
> **출처:** Stanford Law School (뉴스 리포트)
> **URL:** https://law.stanford.edu/press/ai-outperforms-law-professors-in-stanford-law-study/
> **발행:** 2026-06-01

스탠포드 연구팀이 진행한 블라인드 비교 연구에서 법학 교수들은 학생 질문에 대한 답변으로 AI가 만든 응답을 동료 교수의 답변보다 더 자주 선호하는 것으로 나타났습니다(대조 평가에서 AI가 약 75% 우세). 연구진은 계약법 관련 실무형 질문 40개를 만들고, 교수·AI 응답을 익명으로 비교·평가하도록 설계했습니다. 또한 교수들은 AI 응답을 교육적으로 해롭다고 표시할 가능성이 인간 답변보다 낮았다고 보고했습니다.

연구는 AI의 법학 교육 보조 가능성을 시사하지만, 과잉 의존·환각(hallucination)·비판적 사고 약화 등의 위험을 경고하며 '어떻게' 도입할지에 대한 추가 실험과 설계가 필요하다고 결론 내립니다.

**💡 핵심 포인트**
- 블라인드 평가에서 교수들이 AI 답변을 더 높게 평가(약 75% 승률)
- AI 응답이 교육적으로 해롭다고 표시되는 비율이 인간 답변보다 낮음
- 법학처럼 판단·논증이 중요한 분야에서도 LLM의 튜터링 잠재력 확인

**🤔 추가 인사이트**
교육현장에서 AI를 보조 도구로 활용할 때는 성능 우수성 외에 평가·검증 프로세스가 필수입니다. 한국 법학·전문교육 기관은 교육 목표(비판적 사고·윤리 교육 등)에 미치는 영향을 평가하고, AI 보조를 학습 보조·평가 보조로 제한하는 가이드라인을 마련해야 합니다.

### 1. GPU VRAM을 스왑으로 활용하는 실험적 툴 — nbd-vram (GitHub)

> **출처:** GitHub — https://github.com/c0dejedi/nbd-vram

소형·노트북 환경에서 NVidia GPU의 VRAM을 네트워크 블록 장치(NBD)를 통해 스왑 파티션으로 사용하는 오픈소스 프로젝트입니다. 데몬이 VRAM을 할당하고 NBD로 제공하면 커널이 /dev/nbdX로 인식해 스왑으로 사용합니다. 대역폭 측면에서 NVMe보다 빠른 PCIe 경로를 활용해 스왑 성능을 개선하는 아이디어입니다.

왜 주목할까?
- 온디바이스(특히 메모리 업그레이드가 불가한 노트북)에서 메모리 부족을 완화하는 실용적 해법
- 데스크톱/서버·클라우드 GPU 활용 관점에서 새로운 자원 활용 패턴을 제시

*출처: https://github.com/c0dejedi/nbd-vram*
