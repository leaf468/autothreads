### 1. 웹에 에이전트가 침투한다 — 워드프레스, AI 에이전트로 글 작성·발행 허용

> **원제:** WordPress.com now lets AI agents write and publish posts, and more
> **출처:** TechCrunch (Sarah Perez) — https://techcrunch.com/2026/03/20/wordpress-com-now-lets-ai-agents-write-and-publish-posts-and-more/
> **발행:** 2026-03-20

워드프레스닷컴이 AI 에이전트의 사이트 내 글 작성·초안 생성·댓글 관리·메타데이터 수정 등 직접적인 편집 권한을 허용한다고 발표했다. 플랫폼은 MCP(Model Context Protocol)를 통해 에이전트가 사이트 컨텍스트(테마·콘텐츠·설정 등)를 읽고, 사용자가 자연어로 지시하면 자동으로 페이지를 만들거나 초안을 생성하도록 지원한다. 기본 설정은 '초안 저장' 또는 사용자 승인 필요지만, 권한을 부여하면 거의 자동으로 사이트 운영이 가능해진다.

이 변화는 웹 퍼블리싱의 문턱을 낮추는 긍정적 측면이 있지만, 동시에 ‘기계가 만든 콘텐츠’ 폭증과 품질·신뢰성 문제(저작권·사실성·생성물 출처 표기 등)를 야기할 가능성이 크다. 플랫폼은 활동 기록(Activity Log)으로 모든 변경을 추적하지만, 대규모 자동화 시대의 콘텐츠 검증·정책 수립이 새로운 과제가 될 전망이다.

**💡 핵심 포인트**
- WordPress.com은 AI 에이전트에 콘텐츠 작성·발행·댓글 관리 등 실무 권한을 제공
- MCP 지원으로 에이전트가 사이트 컨텍스트를 활용해 일관된 디자인·SEO를 반영 가능
- 기본은 사용자 승인이나, 권한 범위에 따라 자동화된 ‘에이전트 운영’이 현실화됨

**🤔 추가 인사이트**
한국의 미디어·기업 웹사이트 운영자는 에이전트 도입 시 검증 워크플로(사전 승인·표시·감사 로그)를 마련해야 한다. 공공·공신력 있는 정보 유통 채널은 특히 ‘AI 작성 표시’ 규정과 사실 검증 절차를 강화할 필요가 있다.

### 2. 터미널 속 AI 개발자 — OpenCode가 주목받다 (Hacker News 트렌드)

> **원제:** OpenCode – Open source AI coding agent
> **출처:** OpenCode (opencode.ai) · Hacker News — https://opencode.ai/ · https://news.ycombinator.com/item?id=47460525
> **발행:** 2026-03 (사이트/토론)

오픈소스 코딩 에이전트인 OpenCode가 주목받고 있다. 터미널·IDE·데스크탑 앱으로 동작하며 여러 모델(클라우드·로컬)을 연결해 코드 작성, LSP 통합, 멀티세션 운영, 세션 공유 같은 개발자 생산성 기능을 제공한다. 특히 프라이버시 우선 설계(코드·컨텍스트 미저장)와 다양한 모델 연동을 강조해 기업·개발자 커뮤니티에서 빠르게 확산 중이다.

커뮤니티 반응은 활발하다 — HN에서 상단에 오른 이유는 ‘실용적인 에이전트 도구’로서 개발 워크플로를 실질적으로 바꿀 잠재력 때문이다. 오픈소스 기반이라 보안·커스터마이징·내부 호스팅에 유리한 점도 강점으로 꼽힌다.

**💡 핵심 포인트**
- OpenCode는 멀티모델·멀티에디터 통합 코딩 에이전트로 개발 생산성 개선을 목표로 함
- 프라이버시 정책(컨텍스트 미저장)과 다양한 모델 호환성이 채택 이유
- 오픈소스라 기업 내부 도입·검증이 상대적으로 쉬움

**🤔 추가 인사이트**
국내 개발팀은 OpenCode 같은 도구를 평가해 내부 코드 리뷰·CI 파이프라인과 결합하면 보안·생산성 모두에서 이득을 얻을 수 있다. 다만 자동 코드 생성의 품질·라이선스·빌드 안정성 검증은 필수적이다.

### 3. 오디오 MLLM은 ‘진짜 소리’를 이해할까? — DEAF 벤치마크 발표 (arXiv)

> **원제:** DEAF: A Benchmark for Diagnostic Evaluation of Acoustic Faithfulness in Audio Language Models
> **출처:** arXiv — https://arxiv.org/abs/2603.18048
> **발행:** 2026-03-17 (arXiv 제출)

DEAF는 오디오 멀티모달 LLM들이 실제로 음향 신호(감정·배경음·화자 정체성 등)를 잘 해석하는지 진단하는 벤치마크다. 2,700개 이상의 충돌(conflict) 자극을 설계해 텍스트 정보가 모델 예측을 얼마나 편향시키는지 단계적으로 평가한다. 저자들은 여러 Audio MLLM을 평가한 결과, 모델들이 음향 변화에 민감하긴 하지만 최종 예측은 텍스트 컨텍스트에 의해 압도되는 경향(‘텍스트 우위’)을 보였다고 결론지었다.

이는 기존의 음성 벤치마크에서는 드러나지 않던 ‘진정한 음향 이해’의 공백을 드러낸다. 즉, 모델 점수는 높을 수 있지만 음향 신호 자체를 기반으로 합리적 판단을 내리는 능력은 아직 제한적이다.

**💡 핵심 포인트**
- DEAF는 음향적 충돌 사례로 모델의 ‘음향 충실도(acoustic faithfulness)’를 측정함
- 평가 결과, 모델들은 텍스트 힌트에 크게 의존하는 경향을 보였음
- 오디오 이해를 요구하는 실서비스(감정 분석·보안·포렌식 등)에는 추가 검증 필요

**🤔 추가 인사이트**
한국어·다중언어 오디오 서비스를 개발하는 팀은 DEAF 같은 진단 도구를 도입해 ‘텍스트-음향 상호작용’ 취약성을 점검해야 한다. 실제 환경에서는 노이즈·억양·방언이 복합적으로 작용하므로 모델 평가 시 텍스트 유도성(prompt leakage)을 통제하는 실험 설계가 필요하다.

*원문 PDF:* https://arxiv.org/pdf/2603.18048

### 3. Attention Residuals — 깊이(attention-over-depth)로 잔차를 재설계 (GitHub + arXiv)

> **출처:** GitHub · arXiv — https://github.com/MoonshotAI/Attention-Residuals · https://arxiv.org/abs/2603.15031

Attention Residuals(AttnRes)는 트랜스포머의 잔차 연결을 입력-의존적 주의(attention)로 대체해 층 간 정보 흐름을 선택적으로 집계한다. Block AttnRes 같은 실용적 변형은 메모리·성능 균형을 맞추면서 깊은 모델에서의 학습·추론 성능을 개선한다.

왜 주목할까?
- 대형 모델의 스케일링·학습 안정성 문제를 건드리는 구조적 개선으로, 실제 성능 향상 케이스가 보고됨

*출처: https://github.com/MoonshotAI/Attention-Residuals · https://arxiv.org/abs/2603.15031*
