### 1. ProgramBench — 에이전트가 프로그램을 '처음부터' 재구성할 수 있나?

> **원제:** ProgramBench: Can Language Models Rebuild Programs From Scratch?
> **출처:** arXiv
> **링크:** https://arxiv.org/abs/2605.03546
> **발행:** 2026-05-05 (arXiv)

ProgramBench는 LLM 기반 에이전트가 주어진 문서와 테스트만으로 완전한 코드베이스를 설계·구현해 실행 가능한 프로그램을 재구성할 수 있는지를 평가하는 새 벤치마크입니다. 저자들은 다양한 크기·성격의 200개 과제를 준비하고, 에이전트 주도의 퍼징(fuzzing)으로 행동기반 테스트를 생성해 모델을 평가했습니다. 실험 결과 대부분의 모델이 인간 수준의 종합적 소프트웨어 작성 능력에는 크게 못 미치며, 최상위 모델도 소수 과제에서만 높은 통과율을 보였습니다.

ProgramBench는 단편적 버그 수정·기능 추가가 아니라 설계·아키텍처 선택과 통합 테스트를 요구하는 '엔드투엔드' 능력을 측정합니다. 연구는 모델들이 종종 단일 파일·모놀리식 구현에 치중해 인간이 쓴 코드와 구조적으로 차이를 보인다고 지적합니다.

**💡 핵심 포인트**
- 에이전트의 소프트웨어 개발 능력을 종합적으로 평가하는 최초급 벤치마크(200개 과제)
- 대부분의 LLM이 완전한 코드베이스 재구성에서는 성과 미흡
- 모델들이 설계·구조 수준의 결정을 하지 못하고 단편적 구현에 머무르는 경향

**🤔 추가 인사이트**
한국 기업이 에이전트에게 코드 생성·유지보수를 위임하려면 단순 코드 생성 능력뿐 아니라 설계 검증·테스트 자동화·아키텍처 리뷰를 함께 도입해야 합니다. 내부 개발 파이프라인에 에이전트 출력을 통합하려면 행동기반 테스트·CI 파이프라인과의 연동이 필수입니다.

### 2. LongSeeker — 장기 탐색 에이전트를 위한 탄력적 컨텍스트 관리

> **원제:** LongSeeker: Elastic Context Orchestration for Long-Horizon Search Agents
> **출처:** arXiv
> **링크:** https://arxiv.org/abs/2605.05191
> **발행:** 2026-05-06 (arXiv)

LongSeeker는 장기(롱-호라이즌) 탐색·수집 작업을 수행하는 에이전트의 '작업 메모리' 문제를 해결하려고 합니다. 핵심은 모든 중간 산출물을 무작정 누적하지 않고, relevance(관련성)에 따라 세분화(요약·압축·삭제·롤백)하는 Context-ReAct 패러다임을 도입한 것입니다. 논문은 Skip/Compress/Rollback/Snippet/Delete 같은 연산을 정의하고, Compress의 표현력 완비성 등 이론적 보장과 실험적 성능 향상을 제시합니다.

실험에서 LongSeeker는 기존 시스템보다 BrowseComp 등 검색 과제에서 유의미한 성능 개선을 보였고, 컨텍스트를 능동적으로 재구성함으로써 비용 절감과 환각(hallucination) 위험을 낮출 수 있음을 보였습니다.

**💡 핵심 포인트**
- 컨텍스트를 동적으로 압축·선별해 장기 추적 문제 해결
- Compress 연산의 이론적 완비성 및 실용적 비용·정확도 개선
- 장기 플래닝·툴 호출이 많은 에이전트에 적합한 구성

**🤔 추가 인사이트**
국내 서비스에서 장시간 흐르는 자동화(예: 리서치 에이전트, 자동화된 고객 지원)는 컨텍스트 관리 전략이 비용·정확도에 직접적 영향이 큽니다. LongSeeker 같은 접근은 대화 로그·증거 보존 정책을 만족시키면서도 실시간 응답성을 유지하려는 한국 기업에 실용적 대안이 됩니다.

### 3. Curated AI — 제약(Pharma) 분야에서 ‘큐레이티드 인덱스’가 프런티어 LLM을 앞서다

> **원제:** Curated AI beats frontier LLMs at pharma asset discovery
> **출처:** arXiv
> **링크:** https://arxiv.org/abs/2605.04908
> **발행:** 2026-05-06 (arXiv)

이 연구는 일반 웹 검색 기반의 범용 LLM보다는, 도메인·대상별(타깃·표적단위)으로 큐레이션된 인덱스를 사용하는 시스템이 니치한 제약 파이프라인(전임상·아시아권 개발 자산 등)을 찾아내는 데 월등한 성능을 보인다고 보고합니다. 실험에서 'Gosset'이라 불리는 큐레이티드 플랫폼은 동일 질의에 대해 검증된 약물 자산을 프런티어 모델 대비 3.2배 더 많이 찾아냈고, 높은 정밀도와 재현율을 보였습니다.

결론은 단순히 모델 크기·범용성에 의존하기보다, 적절히 구성된 외부 인덱스(도메인 지식·데이터베이스)를 툴로 연결하면 실제 업무 성과이 크게 올라간다는 실무적 메시지입니다.

**💡 핵심 포인트**
- 큐레이티드 인덱스(도메인 전용 데이터)는 범용 LLM+웹검색을 능가할 수 있음
- 제약·의료 같은 롱테일 정보가 중요한 분야에서 특히 효과적
- 프런티어 모델에도 동일 인덱스를 툴로 연결하면 성능 격차 대부분 해소 가능

**🤔 추가 인사이트**
한국의 바이오·제약 스타트업은 내부·협력사 데이터의 품질을 높이고, 이를 모델의 '툴'로 노출하는 전략(예: 내부 레지스트리, 임상·특허 DB 연동)을 통해 AI 기반 스카우팅·시드 발굴 효율을 빠르게 개선할 수 있습니다.

### 1. DeerFlow — 장기·서브에이전트 중심 SuperAgent 하니스 (GitHub Trending)

> **출처:** GitHub (bytedance/deer-flow)
> **링크:** https://github.com/bytedance/deer-flow

DeerFlow는 서브에이전트, 샌드박스, 메모리, 스킬을 결합한 오픈소스 '슈퍼에이전트 하니스'입니다. 설치 가이드·채널 연동·샌드박스 실행 등 실전 배포용 기능을 풍부하게 제공해 리서치·자동화 워크플로우를 빠르게 프로덕션화할 수 있게 설계돼 있습니다.

왜 주목할까?
- 장기·병렬화된 작업을 서브에이전트로 분산해 실제 업무 자동화에 근접한 설계
- LangGraph/LangChain 기반으로 확장성과 통합성 우수

원문: https://github.com/bytedance/deer-flow

### 2. Dexter — 금융 리서치 특화 자율 에이전트 (GitHub Trending)

> **출처:** GitHub (virattt/dexter)
> **링크:** https://github.com/virattt/dexter

Dexter는 금융 리서치에 특화된 자율 에이전트로, 실시간 시장 데이터·테스크 플래닝·자가검증 루프를 통해 깊이 있는 분석을 자동으로 수행합니다. 금융 도메인에 맞춘 툴체인·평가 파이프라인을 포함합니다.

왜 주목할까?
- 실무 금융 리서치 워크플로우를 자동화하려는 시도는 기업 적용 가능성 높음
- 한국의 퀀·애널리스트 팀이 내부 데이터와 결합하면 즉시 활용 가능

원문: https://github.com/virattt/dexter

### 3. 블로그: "Vibe coding and agentic engineering are getting closer" — Simon Willison

> **출처:** Simon Willison weblog
> **링크:** https://simonwillison.net/2026/May/6/vibe-coding-and-agentic-engineering/

Simon Willison은 ‘vibe coding’(비숙련자가 에이전트에게 맡겨 결과만 사용하는 방식)과 전문 엔지니어가 툴로 에이전트를 활용하는 ‘agentic engineering’ 사이의 경계가 점점 모호해지고 있다고 진단합니다. 특히 코드 검토를 완전히 생략하는 관행의 위험성, 에이전트 의해 생성된 프로젝트의 평가·신뢰 문제, 에이전트 도입이 바꾸는 개발 라이프사이클 병목 등을 짚습니다.

왜 주목할까?
- 실무에서 에이전트 신뢰성·책임 소재 문제를 고민하는 엔지니어 관점의 실용적 통찰
- 한국 조직의 거버넌스·QA 정책 설계에 참고될 만한 인사이트
