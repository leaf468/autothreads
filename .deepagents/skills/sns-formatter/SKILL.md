# SNS 포맷터 스킬

이 스킬은 기존 AI 트렌드 마크다운 리포트를 Threads 및 X(Twitter) 소셜 미디어 스타일로 변환합니다.

## 입력

- **원본 파일**: `reports/YYYYMMDD-HHMM.md` (기존 마크다운 리포트)

## 출력

### 1. Threads 버전
- **파일명**: `reports/YYYYMMDD-HHMM-threads.txt`
- **스타일**: 텍스트 중심, 가벼운 톤, 이모지 활용

### 2. X(Twitter) 스레드 버전
- **파일명**: `reports/YYYYMMDD-HHMM-twitter.txt`
- **스타일**: 280자 제한, 스레드 형식, 해시태그

---

## Threads 포맷팅 규칙

### 스타일 가이드
1. **톤앤매너**
   - 캐주얼하고 친근한 어투
   - "~요", "~네요" 등 부드러운 종결어미
   - 완벽한 보고서보다 솔직한 공유 느낌

2. **구조**
   - 짧은 문단 (3-5문장)
   - 각 섹션은 공백으로 구분
   - 이모지를 적절히 활용 (과하지 않게)

3. **이미지 안내**
   - 세로형 이미지 권장 (4:5 비율)
   - 원본 URL은 첫 댓글에 배치 권장

4. **링크 전략**
   - 본문에는 링크 최소화
   - 주요 링크는 "댓글에 링크 남겼어요!" 형식으로 안내


## X(Twitter) 스레드 포맷팅 규칙

### 스타일 가이드
1. **톤앤매너**
   - 간결하고 임팩트 있게
   - 핵심만 전달
   - 정보 밀도 높게

2. **구조**
   - 첫 트윗: 훅(Hook) - 주목 끌기
   - 중간 트윗: 각 뉴스 1트윗
   - 마지막 트윗: 요약 + 링크

3. **280자 제한**
   - 각 트윗은 280자 이내
   - 필요시 줄임말, 기호 활용
   - 링크는 별도 카운트

4. **해시태그**
   - 2-3개 핵심 해시태그
   - 트렌딩 태그 활용

## 실행 순서

1. **원본 파일 읽기**
   ```
   ls reports/*.md | tail -1
   cat reports/YYYYMMDD-HHMM.md
   ```

2. **Threads 버전 생성**
   - 원본 내용을 Threads 스타일로 변환
   - `reports/YYYYMMDD-HHMM-threads.txt` 저장

3. **X(Twitter) 스레드 버전 생성**
   - 원본 내용을 Twitter 스레드로 변환
   - 각 트윗 280자 체크
   - `reports/YYYYMMDD-HHMM-twitter.txt` 저장

---

## 주의사항

1. **원본 유지**: 기존 마크다운 파일은 절대 수정하지 않음
2. **캐릭터 카운트**: Twitter는 280자 엄격히 준수
3. **이모지 절제**: 과하지 않게, 가독성 우선
4. **링크 처리**:
   - Threads: "댓글 참고" 안내
   - Twitter: 마지막 트윗에만 포함
5. **한글 자연스러움**: 번역체 금지, 자연스러운 한국어
